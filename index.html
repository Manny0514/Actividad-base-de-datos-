Conceptos de Datos y Bases de Datos

1. ¿Qué es un Dato?

Un dato es una representación simbólica de una realidad o evento, que puede ser cuantitativo o cualitativo. Los datos pueden ser números, letras, imágenes, sonidos, etc., que en su forma aislada no tienen significado por sí mismos hasta que se interpretan en un contexto específico.

2. ¿Qué es información?

La información es un conjunto de datos procesados, organizados y contextualizados de tal manera que adquieren significado y son útiles para la toma de decisiones. La información proporciona conocimiento y comprensión sobre un tema o situación.

3. Diferencia entre dato e información

Dato: Es una representación cruda y sin procesar de hechos o conceptos.

Información: Es el resultado de procesar y organizar datos de manera que se vuelvan útiles y significativos.

4. Tipos de datos

  • Datos cualitativos: Describen cualidades o características y no se pueden medir con números (ej. color, textura).
  • Datos cuantitativos: Se pueden medir y expresar con números (ej. altura, peso).
  • Datos estructurados: Tienen un formato predefinido y organizado (ej. tablas en una base de datos).
  • Datos no estructurados: No tienen un formato específico ni organizado (ej. correos electrónicos, videos).

5. ¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear un modelo para estructurar y organizar los datos de manera que representen correctamente la realidad del negocio o dominio de interés. Este modelo puede ser conceptual, lógico o físico y sirve como un mapa que guía el diseño y la implementación de sistemas de bases de datos.

6. Tipos de bases de Datos, según:

Variabilidad

  • Bases de datos estáticas: Los datos no cambian con el tiempo o cambian muy poco.
  • Bases de datos dinámicas: Los datos se actualizan y cambian con frecuencia.

Contenido

  • Bases de datos bibliográficas: Contienen referencias a documentos o publicaciones.
  • Bases de datos de texto completo: Contienen el texto completo de los documentos.
  • Bases de datos numéricas: Contienen datos numéricos o cuantitativos.
  • Bases de datos multimedia: Contienen imágenes, audio, video, etc.

Modelo de base de datos

  • Bases de datos jerárquicas: Organizadas en una estructura de árbol.
  • Bases de datos de red: Estructura más flexible que permite múltiples relaciones entre datos.
  • Bases de datos relacionales: Organizadas en tablas con relaciones definidas entre ellas.
  • Bases de datos orientadas a objetos: Almacenan datos como objetos, similar a la programación orientada a objetos.
  • Bases de datos NoSQL: Diseñadas para manejar grandes volúmenes de datos no estructurados.

7. ¿Qué es el modelo relacional?

El modelo relacional es un enfoque para gestionar bases de datos en el cual los datos se organizan en tablas (relaciones). Cada tabla está compuesta por filas (tuplas) y columnas (atributos). Las tablas pueden estar relacionadas entre sí mediante llaves primarias y foráneas, facilitando la integridad y consistencia de los datos.

8. Describa los elementos del Diagrama entidad Relacional

  • Entidad: Un objeto del mundo real o un concepto que tiene existencia independiente y se puede identificar de manera única.
  • Atributo: Una característica o propiedad de una entidad.
  • Relación: Una asociación entre dos o más entidades.
  • Llave primaria: Un atributo o conjunto de atributos que identifica de manera única a una entidad en una tabla.
  • Llave foránea: Un atributo en una tabla que hace referencia a la llave primaria de otra tabla para establecer una relación entre las dos.

9. ¿Qué es una entidad?

Una entidad es cualquier objeto o concepto que puede ser identificado de manera única en un dominio de datos. Puede ser una persona, un lugar, un objeto, un evento o incluso un concepto abstracto.

10. ¿Qué es un atributo y sus tipos?

Un atributo es una característica o propiedad de una entidad. Los tipos de atributos pueden incluir:

  • Atributo simple: No se puede dividir en subcomponentes más pequeños (ej. número de teléfono).
  • Atributo compuesto: Puede dividirse en subcomponentes (ej. dirección: calle, ciudad, código postal).
  • Atributo derivado: Su valor se calcula a partir de otros atributos (ej. edad calculada a partir de la fecha de nacimiento).
  • Atributo multivaluado: Puede tener múltiples valores (ej. números de teléfono de una persona).

11. ¿Qué es una relación?

Una relación es una asociación entre dos o más entidades que define cómo están conectadas. Por ejemplo, una relación puede indicar que un empleado trabaja en un departamento o que un cliente realiza un pedido.

12. ¿Qué son las llaves primarias, foránea y sus características?

Llave primaria: Un atributo o conjunto de atributos que identifica de manera única a una entidad en una tabla. Debe ser única, no nula y constante.

Llave foránea: Un atributo en una tabla que es la llave primaria en otra tabla. Establece una relación entre las dos tablas y garantiza la integridad referencial.

13. ¿Qué son las reglas del Negocio?

Las reglas del negocio son restricciones, condiciones o criterios que definen o restringen aspectos del negocio. Guían el comportamiento del sistema y aseguran que los datos sean consistentes y válidos de acuerdo con las políticas y objetivos del negocio.

14. ¿Qué es la cardinalidad o mapeo?

La cardinalidad o mapeo define el número de instancias de una entidad que pueden o deben estar asociadas con una instancia de otra entidad en una relación. Los tipos comunes de cardinalidad incluyen:

  • Uno a uno (1:1): Cada instancia de una entidad se asocia con una instancia de otra entidad.
  • Uno a muchos (1:N): Cada instancia de una entidad se asocia con múltiples instancias de otra entidad.
  • Muchos a muchos (M:N): Múltiples instancias de una entidad se asocian con múltiples instancias de otra entidad.
